# Acceptance Criteria: <기능명>

> Phase 0에서 작성. Phase 4에서 그대로 E2E 테스트가 된다.
> 관찰 가능한 결과만 Then에 쓴다(구현 디테일 금지). 성공 + 핵심 실패 경로를 모두 덮는다.

## 시나리오 1 (성공): <제목>

```
Given <전제 조건>
  And <추가 전제>
When  <트리거하는 행위>
Then  <관찰 가능한 결과>
  And <추가 결과>
```

## 시나리오 2 (실패): <제목, 예: 재고 부족>

```
Given <전제>
When  <행위>
Then  <거부/예외 결과>
  And <부수효과가 없어야 함, 예: 저장 안 됨>
```

## 시나리오 3 (실패/경계): <제목>

```
Given ...
When  ...
Then  ...
```
